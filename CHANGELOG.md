Markdown chapter breaks follow the heading hierarchy ‚Äî chapter level = shallowest heading level that repeats; a lone leading `#` title is treated as front matter; deeper headings become in-chapter section headings (nested reader TOC). ### Changed
- **Universal Mac app (one App Store record, buy once):** the macOS target's bundle id is unified to `com.echo.audiobooks` (was `com.echo.audiobooks.macos`), so Echo ships as a single universal app across iPhone + Mac instead of a separate Mac record. - **Narration engine swapped to ONNX Runtime (CPU) ‚Äî instant start, every device:** On-device Kokoro narration now runs through `OnnxKokoroEngine` (ONNX Runtime, the `model_fp16.onnx` graph from `onnx-community/Kokoro-82M-v1.0-ONNX`, ~163 MB) on both iOS and macOS, replacing the CoreML stack. The CoreML path (FluidAudio's `KokoroTTSEngine`, the later fixed-shape `KokoroFixedShapeEngine`, and the `NarrationModelStore` 731 MB model downloader) AOT-compiled its model graphs on-device on first run ‚Äî ~20 minutes on an A14 iPhone, because the LSTM duration predictor's Espresso compile is O(n¬≤) in token length ‚Äî and routed the vocoder onto the Neural Engine, which trapped (`libBNNS` `EXC_BREAKPOINT`) on A14. ### Fixed
- **Narration: multi-second silent gaps eliminated (the model's all-zero output is caught and recovered):** The Kokoro ONNX model intermittently returns a full-length but **all-zero** waveform for a chunk ‚Äî landing as pure digital silence in the narrated audio (measured at ~10‚Äì12% of chunks on *Git Happens*, some clustering into gaps up to 24 s, ~20% of the book overall). `OnnxKokoroEngine` now routes every synthesis through `NarrationSilenceGuard`, which detects a silent (non-empty, near-zero-peak) result and recovers ‚Äî first re-running with a perturbed input (a trailing space changes the token ids; the CPU EP is deterministic, so an identical re-run would just repeat the zero), then splitting the text at a word boundary ‚Äî so a real word is never dropped to silence. A boundary-only (`[BOS, EOS]`) id sequence is now treated as "nothing to say" rather than fed to the model. New `NarrationSilenceGuardTests`. (iOS + macOS)
- **Narration: hyphenated compounds read smoothly instead of with a hard pause.** A plain hyphen (`rough-and-ready`, `save-as`, `open-source`) was tagged as a dash and rendered as the long em-dash pause `‚Äî`, so the narrator said "rough _[pause]_ and _[pause]_ ready". A hyphen now reads as a brief word break; genuine em/en dashes still pause, and a *spaced* dash used as a sentence pause (`a ‚Äî b`, `a ‚Äì b`, `a - b`) is normalized to a comma so it keeps its pause. New `HyphenJoinerTests` + `TextNormalizer` cases.
- **Narration: an unknown word is never silently skipped.** Any letter-bearing word the lexicon and OOV fallback leave unvoiced is now spelled out via the deterministic grapheme‚ÜíIPA approximator at the G2P's final assembly, closing the legacy path where the `‚ùì` marker reached `KokoroPhonemeVocab` and was dropped to silence. New `VoicelessGuaranteeTests`.
- **Narration: a pronunciation-override link is no longer torn apart by chunking.** `[word](/ipa/)` overrides are applied before the ‚â§200-char chunker, and an IPA syllable separator `.` was triggering a sentence split that inserted spaces inside the link and broke the override.
